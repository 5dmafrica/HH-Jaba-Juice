<analysis>**original_problem_statement:**
The user wants to build a HH Jaba Staff Portal, a mobile-first internal ordering system for employees of 5DM Africa.

**PRODUCT REQUIREMENTS:**
- **Technology Stack:** React, FastAPI, and MongoDB.
- **Multi-Role Authentication:**
    - **User:** Standard employee with  email. Can place orders, view history, submit payments, and raise disputes.
    - **Admin:** Manages orders, stock, users, and payments.
    - **Super Admin ():** Has full admin rights plus the ability to switch roles (impersonate Admin or User for testing), bypass order/credit limits, and access a maintenance tab to reset test data.
- **Product Catalog:** Manage a list of flavored drinks with stock and pricing.
- **Ordering System:**
    - Users have a daily order limit (10 bottles) and a monthly credit limit (KES 30,000).
    - All new orders default to 'Pending' status.
- **Payment & Credit Management:**
    - A No Carry-Forward rule applies to monthly credit balances.
    - A consolidated Invoices module handles both credit and cash transactions.
    - Customers submit text-based Proof of Payment (POP) containing a transaction code and amount against specific invoices.
    - **Transaction Matching:** An order is only fulfilled after an Admin manually enters matching transaction details (code, amount) to the customer's POP submission.
    - **Dispute Resolution:** If a payment match fails, the status becomes 'Verification Failed', and the customer can initiate a chat dispute. Admins have a central 'Messages' hub to manage and resolve these disputes, with the ability to 'Force Approve' a transaction.
- **Admin Dashboard:** A multi-tab dashboard for managing pending orders, payments (verification queue), stock, user reconciliation, defaulters, chat messages, and maintenance (for Super Admin).
- **Notifications & Sharing:** Admins can push offers to users. Invoices can be shared via WhatsApp/Email. Users receive in-app notifications for status changes, and invoices have a 'Read/Unread' status.

**User's preferred language**: English

**Key Technical Concepts**

- **Backend:** FastAPI (Python), Motor (async driver for MongoDB)
- **Database:** MongoDB

**key DB schema**
- **users:** { user_id, email, name, role: ('user' | 'admin' | 'super_admin'), credit_balance, ... }
- **products:** { product_id, name, price, stock, ... }
- **orders:** { order_id, user_id, items, total_amount, status: ('pending' | 'fulfilled' | 'cancelled'), ... }
- **credit_invoices:** { invoice_id, user_id, line_items, total_amount, status: ('unpaid' | 'paid'), ... }
- **pop_submissions:** { pop_id, invoice_id, user_id, transaction_code, amount_paid, status: ('pending' | 'approved' | 'verification_failed'), ... }
- **disputes:** { dispute_id, pop_id, user_id, messages: [{ sender, text, timestamp }], status: ('open' | 'resolved'), ... }
- **notifications:** { notification_id, user_id, message, read, ... }
